# Development Guide

AIOSS의 표준 개발 워크플로우를 설명합니다.

## 브랜치 전략

1. main 브랜치에서 기능 브랜치를 생성합니다.
2. 커밋은 작고 명확한 단위로 남깁니다.
3. 피드백을 위해 PR을 빠르게 열어둡니다.

## 개발 절차

1. main의 최신 변경사항을 가져옵니다.
2. 하나의 논리적 변경 단위로 구현합니다.
3. 로컬 점검과 테스트를 수행합니다.
4. 의미 있는 커밋 메시지로 커밋합니다.
5. 푸시 후 PR을 생성하거나 업데이트합니다.

## 문서화 절차

- 동작이 바뀌면 README 또는 Wiki를 함께 업데이트합니다.
- 반복되는 이슈는 Troubleshooting에 해결 방법을 추가합니다.

# WBS (Work Breakdown Structure) — LightGauss Optimizer

### 1. 프로젝트 준비
1.1 요구사항 정리  
1.2 데이터/모델 수집  
1.3 개발/평가 환경 셋업  
1.4 팀 역할/일정 확정  

---

### 2. 모델 분석 및 기반 설계
2.1 GPS‑Gaussian 모델 구조 분석  
2.2 메모리/계산 병목 파악  
2.3 프루닝·샘플링 설계 방향 수립  
2.4 성공 기준(KPI) 정의 및 측정 계획 수립  

---

### 3. Gaussian Pruning 모듈 개발
3.1 중요도 기반 프루닝 알고리즘 설계  
3.2 임계값 결정 전략 연구 (오픈 이슈 #1)  
3.3 프루닝 구현  
3.4 프루닝 품질/메모리 감소 검증 (≥30% 감소)  
3.5 후처리(보정 필터) 설계 및 적용  

---

### 4. 중요도 기반 Sampling 모듈 개발
4.1 디테일 영역 정의 및 마스크 설계  
4.2 렌더링 기여도/오차 기반 샘플링 알고리즘 설계  
4.3 중요도 점수(0~1) 계산식 정의 (오픈 이슈 #2)  
4.4 샘플링 구현  
4.5 재현율(≥90%) 검증  

---

### 5. 통합 파이프라인 구축
5.1 프루닝 + 샘플링 통합 워크플로우 설계  
5.2 CLI/스크립트 (단일 명령 실행) 구현  
5.3 파이프라인 재실행 재현성 확보 (비기능 요구사항)  
5.4 로그/오류 메시지 정비  
5.5 모델 버전 관리/출력 포맷 정리  

---

### 6. 평가 & 벤치마크
6.1 성능 측정 도구 구현 (메모리/속도)  
6.2 시각 품질 측정 도구 구현 (PSNR/SSIM 자동 보고)  
6.3 벤치마크 스위트 (그래프/CSV 저장)  
6.4 평가 데이터셋 준비 (100개 씬, 수백만 가우시안)  
6.5 오프라인 평가 (KPI 기준 검증)  
6.6 디바이스 파일럿 테스트 계획 (VR/모바일)

### 7. 문서화 및 릴리스 준비

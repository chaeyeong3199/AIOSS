# Contributing Guide

AIOSS 저장소에 기여해 주셔서 감사합니다.

## 1. 작업 방식

1. 이슈를 확인하거나 새 이슈를 등록합니다.
2. `main`에서 기능 브랜치를 생성합니다.
   - 예: `feature/add-l06-docs`, `fix/metrics-parser`
3. 작은 단위로 변경하고 의미 있는 커밋 메시지를 작성합니다.
   - 권장: Conventional Commits (`feat:`, `fix:`, `docs:` 등)
4. Pull Request(PR)를 생성하고 변경 이유/검증 방법을 작성합니다.

## 2. 코드 및 문서 규칙

- 기존 디렉터리 구조와 파일 네이밍을 유지합니다.
- 스크립트 변경 시 실행 방법을 문서에 함께 반영합니다.
- README/과제 문서는 한국어 중심으로, 필요한 경우 영어 용어를 병기합니다.

## 3. Pull Request 체크리스트

PR 생성 전 아래 항목을 확인해 주세요.

- [ ] 변경 목적이 PR 본문에 명확히 작성되었는가
- [ ] 관련 문서(README/과제 문서)를 필요한 만큼 갱신했는가
- [ ] 로컬에서 실행/검증을 완료했는가
- [ ] 민감 정보(API 키, 토큰, 개인 정보)가 포함되지 않았는가

## 4. 리뷰 규칙

- 필수 수정 요청은 `[MUST]`로 표기합니다.
- 개선 제안은 `[SHOULD]`로 표기합니다.
- 사람보다 코드와 근거에 집중하여 존중하는 톤으로 리뷰합니다.

## 5. 문의

질문이나 제안은 GitHub Issues 또는 Discussions를 통해 남겨 주세요.

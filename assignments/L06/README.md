## 🧭 Inner Source 운영 계획(초안)

1. 협업 단위
	- 기능/문서 변경은 이슈 기반으로 등록 후 브랜치에서 작업
2. 검토 기준
	- 리뷰 코멘트는 `[MUST]`, `[SHOULD]` 태그로 구분
3. 문서 동기화
	- 스크립트 또는 운영 규칙 변경 시 README/과제 문서 동시 업데이트
4. 품질 게이트
	- PR 본문에 변경 이유, 검증 방법, 영향 범위를 필수 기입

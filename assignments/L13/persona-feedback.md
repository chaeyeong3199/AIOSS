# Week 13 Persona Feedback

## 피드백 수집 방식

본 과제에서는 LLM을 활용해 서로 다른 사용 패턴을 가진 10명의 가상 사용자를 생성했다. 각 사용자는 AIOSS 대시보드에서 A/B 테스트 variant를 확인하고 CTA 버튼을 클릭하는 시나리오를 수행한 뒤, 만족도와 개선 의견을 남겼다.

## 정성 피드백 요약

- Variant B는 실험 목적과 CTA가 더 명확하다는 평가가 많았다.
- Variant A는 기본적인 정보 전달은 가능하지만, 사용자가 “무엇을 해야 하는지”를 바로 이해하기에는 약간 부족했다.
- 일부 사용자는 CTA 버튼의 이름을 더 구체적으로 바꾸면 좋겠다고 제안했다.
- DevOps 관점에서는 Feature Flag와 Canary Rollout이 실험 운영 구조와 잘 연결되어 있다는 평가를 받았다.
- QA 관점에서는 `/api/flags`, `/api/track`, `/api/health`가 함께 검증되어 실험 추적 흐름이 명확하다는 의견이 있었다.
